# ComponentOnlyValidator

**Package:** com.hypixel.hytale.server.npc.asset.builder.validators
**Type:** Singleton

## Definition
```java
// Signature
public class ComponentOnlyValidator extends Validator {
```

## Architecture & Concepts
The ComponentOnlyValidator is a specific, stateless rule implementation within the server's NPC asset validation framework. Its sole purpose is to enforce a structural constraint: that a given asset definition or a subsection thereof must contain *only* component data and no other asset types like models, textures, or sounds.

Architecturally, this class embodies the **Strategy Pattern** for validation. It is a concrete, non-configurable strategy that can be plugged into a larger validation engine or asset builder. The use of the **Singleton Pattern** is a critical optimization; since the validator is stateless, a single shared instance can be used across the entire server to validate any number of assets, eliminating the overhead of repeated object instantiation. It acts as a gate in the data processing pipeline for NPC creation.

## Lifecycle & Ownership
- **Creation:** The single `INSTANCE` is created statically by the JVM class loader when the ComponentOnlyValidator class is first accessed. Its lifecycle is not managed by any application-level factory or dependency injection framework.
- **Scope:** Application-wide. The singleton instance persists for the entire lifetime of the server process.
- **Destruction:** The instance is destroyed only when the JVM unloads the class, typically during a full server shutdown. No manual cleanup is necessary or possible.

## Internal State & Concurrency
- **State:** This class is **stateless and immutable**. It contains no instance fields and its behavior never changes after initialization.
- **Thread Safety:** This class is inherently **thread-safe**. As a stateless singleton, it can be safely referenced and used by multiple threads concurrently without any need for locks or synchronization primitives.

## API Surface
The public contract is minimal, designed for retrieval rather than direct interaction. The core validation logic is inherited from the parent `Validator` class and is not part of this class's direct surface.

| Symbol | Type | Complexity | Description |
| :--- | :--- | :--- | :--- |
| get() | ComponentOnlyValidator | O(1) | Retrieves the shared, global instance of the validator. |

## Integration Patterns

### Standard Usage
This validator is not intended to be used in isolation. It is designed to be passed to a higher-level system, such as an asset builder or a validation runner, which will then invoke the inherited validation logic.

```java
// Hypothetical usage within an NPC asset building process
NpcAssetBuilder builder = new NpcAssetBuilder(assetData);

// The builder uses the validator to check a specific part of the asset
builder.validateSection("component_overrides", ComponentOnlyValidator.get());

if (builder.hasErrors()) {
    // Handle validation failure
}
```

### Anti-Patterns (Do NOT do this)
- **Direct Instantiation:** The constructor is private. Do not attempt to create an instance using reflection. This violates the singleton contract and provides no benefit. Always use `ComponentOnlyValidator.get()`.
- **Stateful Subclassing:** Do not subclass ComponentOnlyValidator to add state. The validation framework expects validators to be stateless and reusable.

## Data Pipeline
ComponentOnlyValidator acts as a conditional gate within the broader NPC asset loading pipeline. It receives a portion of an asset definition and either allows the pipeline to continue or halts it by reporting an error.
